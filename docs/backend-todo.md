# ONDA 관리자 웹 백엔드 구현 상태와 추가 예정 목록

이 문서는 현재 관리자 웹 프론트가 실제 서비스가 되기 위해 필요한 백엔드 작업을 정리한 것입니다. 현재 프론트는 Mock 모드를 기본으로 유지하되, HTTP 모드에서는 Spring Boot API 주요 기능을 호출합니다. 구현되지 않은 기능은 화면에서 `아직 구현 안 됨 · 추가 예정`으로 표시합니다.

## 1. 인증/인가 추가 예정

- 운영용 refresh token/session 만료 정책
- 역할 구분 고도화

  - 시설관리팀
  - 슈퍼관리자
- 역할별 메뉴 접근 제어의 서버 검증
- 역할별 버튼 권한 제어의 서버 검증
- 관리자 활동 감사 로그 조회/내보내기 고도화

## 10. 백엔드 1차 구현 순서

프론트가 이미 Mock으로 완성되어 있으므로, 백엔드는 모든 기능을 한 번에 붙이기보다 관리자 웹에서 가장 많이 쓰는 흐름부터 연결하는 것이 좋습니다. 1차 연결은 일부 완료되었고, 아래 표는 다음 백엔드 확장 순서입니다.

| 순서 | 구현 범위 | 이유 |
|---:|---|---|
| 1 | 운영 DB 연결과 seed 데이터 영속화 | 현재 seed 기반 데이터를 실제 DB로 유지 |
| 2 | 경험 피드 원문/공개본/검수 상태 저장 | 특정 가능성 방지와 공개 검수의 기반 |
| 3 | 개선 워크플로우 과제 생성/단계 변경 | 제보를 학교 개선 업무로 전환 |
| 4 | 중복 제보 병합, 개선 과제 연결 | 반복 문제 분석과 학교 개선 근거 강화 |
| 5 | 파일 업로드/다운로드 고도화 | 제보 사진, 공문, 시설팀 답변서의 검증 강화 |
| 6 | 공공데이터 미설정 endpoint 보강 | seed 데이터가 아닌 실제 data.go.kr 동기화 확대 |
| 7 | 월간 리포트 PDF/CSV 템플릿 고도화 | 학교 제출용 자료 자동화 품질 강화 |
| 8 | 운영 인증/인가 고도화 | 실제 서비스 전 보안 강화 |

## 11. 프론트 연결 우선순위

| 우선순위 | 프론트 함수 | 백엔드 endpoint 후보 | 연결 효과 |
|---:|---|---|---|
| 1 | `getStoriesForAdmin(query?)` | `GET /api/admin/stories` | 경험 피드 검수 목록 연결 |
| 2 | `updateStoryVisibility(id, visibility)` | `PATCH /api/admin/stories/{storyId}/visibility` | 공개/비공개 검수 저장 |
| 3 | `getImprovementTasks(query?)` | `GET /api/admin/improvement-tasks` | 개선 워크플로우 실제 과제 연결 |
| 4 | `updateImprovementTaskStage(id, stage)` | `PATCH /api/admin/improvement-tasks/{taskId}/stage` | 과제 단계 변경 저장 |
| 5 | `getMonthlyReport()` | `GET /api/admin/monthly-report` | 실제 집계 리포트 연결 |
| 6 | CSV 내보내기 | `GET /api/admin/monthly-report/export/csv?yearMonth=YYYY-MM` | 제출 자료 다운로드 |
| 7 | PDF 내보내기 | `GET /api/admin/monthly-report/export/pdf?yearMonth=YYYY-MM` | 연결 완료 |
| 8 | 첨부파일 | `POST /api/admin/reports/{id}/attachments` | 현장 사진/공문 저장 |
| 9 | 공공데이터 동기화 | batch/admin endpoint | data.go.kr 실제 키 기반 갱신 |

- 운영 DB 영속화와 migration
- 상태 변경 사유, 담당 부서, 해결 체크리스트 저장
- 관리자 처리 이력과 감사 로그 검색/필터 고도화
- 경험 피드 원문과 공개본 분리 저장
- 익명화 결과와 관리자 검수 결과 저장
- 첨부파일 검증, 저장 경로 정책, 악성 파일 검사

- 월간 리포트 PDF 템플릿 품질, CSV 스냅샷 검색, 다운로드 이력 UI 고도화

## 13. 2차 이후로 보류할 항목

아래 기능은 공모전 MVP 이후로 미뤄도 됩니다.

- 실시간 위치 기반 도움 요청 push 알림
- 사진 기반 단차/경사 자동 판별
- 실내 정밀 내비게이션
- 실제 LLM API 기반 익명화 자동화는 필요성과 보안 검토 후 판단
- 공공데이터 대량 배치와 장애물 자동 매칭
- PDF 공문 템플릿 자동 생성
- 기관별 다중 캠퍼스 권한 관리

## 14. 민감정보/위치정보/장애 관련 정보 보호

관리자 웹이 실제 서비스가 되면 장애 관련 정보, 위치정보, 경험 피드 원문을 다룰 수 있습니다. 따라서 백엔드 구현 전 아래 항목을 설계에 포함해야 합니다.

- 장애 유형, 도움 요청 위치, 경험 피드 원문은 최소 권한 관리자만 조회
- 경험 피드는 `originalContent`와 `publicContent`를 분리 저장
- 원문 조회, 공개 처리, 다운로드, 파일 열람은 모두 `audit_logs`에 기록
- 위치정보는 보관 기간과 삭제 정책을 명확히 설정
- 운영 로그에 원문, 위치 상세, 사용자 식별 정보를 그대로 남기지 않음
- 첨부파일은 확장자, MIME 타입, 용량 제한, 악성 파일 검사를 적용
- 실제 AI API 연결 전에는 민감정보 전송 범위와 보관 정책을 별도 보안 리뷰
- `.env`, DB password, API Key, Oracle credential, wallet은 GitHub 업로드 금지
