# ONDA 관리자 웹 백엔드 추가 필요 목록

이 문서는 현재 관리자 웹 프론트가 실제 서비스가 되기 위해 필요한 백엔드 작업을 정리한 것입니다. 현재 프론트는 Mock 데이터만 사용합니다.

## 1. 인증/인가

- 관리자 로그인 API
- 역할 구분
  - 장애학생지원센터
  - 시설관리팀
  - 슈퍼관리자
- 역할별 메뉴 접근 제어
- 역할별 버튼 권한 제어
- 관리자 활동 감사 로그 저장

## 2. 접근성 제보 관리

- 제보 목록/상세 조회
- 제보 상태 변경
- 담당자 배정
- 우선순위 변경
- 처리 메모 저장
- 상태 변경 사유 저장
- 담당 부서 배정
- 해결 체크리스트 저장
- 제보 사진/첨부파일 업로드
- 제보 처리 이력 저장

## 3. 도움 요청 관리

- 도움 요청 목록/상세 조회
- 도움 요청 상태 변경
- 응답자 배정
- 센터 판단 저장
- 접근성 제보 연결
- 미응답 요청 알림
- 요청 종료 이력 저장

## 4. 경험 피드 및 AI 익명화

- 경험 피드 원문 저장
- 공개본 저장
- AI 익명화 요청/결과 저장
- 민감정보 탐지 결과 저장
- 관리자 검수 상태 저장
- 신고/비공개 처리
- 원문 접근 권한 제한

## 5. 공공데이터 연동

- 공공데이터 출처 메타데이터 저장
- 전국 장애인 편의시설 표준데이터 연동
- 대학/시설 관련 데이터 연동
- 건축물대장 또는 시설 데이터 연동
- 데이터 갱신 시각 저장
- 현장 제보와 공공데이터 불일치 계산

## 6. 개선 워크플로우

- 개선 과제 생성
- 제보 여러 건을 하나의 개선 과제로 묶기
- 단계 변경
- 담당 부서 배정
- 마감일 지정
- 완료 증빙 파일 저장
- 시설팀 전달 이력 저장

## 7. 월간 리포트

- 월별 데이터 스냅샷 생성
- 건물별 제보/공감/해결률 집계
- 고위험 미해결 항목 계산
- PDF 생성
- CSV 생성
- 공문 번호 발급
- 생성된 리포트 다운로드 이력 저장

## 8. DB 설계 후보

| 테이블 | 설명 |
|---|---|
| `admin_users` | 관리자 계정 |
| `admin_roles` | 관리자 역할 |
| `campuses` | 학교 |
| `buildings` | 건물 |
| `accessibility_reports` | 접근성 제보 |
| `report_notes` | 제보 처리 메모 |
| `report_attachments` | 제보 첨부파일 |
| `help_requests` | 도움 요청 |
| `stories` | 경험 피드 |
| `story_reviews` | AI 익명화/관리자 검수 |
| `public_data_sources` | 공공데이터 출처 |
| `public_data_snapshots` | 공공데이터 동기화 스냅샷 |
| `improvement_tasks` | 개선 과제 |
| `monthly_reports` | 월간 리포트 |
| `audit_logs` | 관리자 감사 로그 |

## 9. 우선순위

| 우선순위 | 작업 |
|---|---|
| 1 | 인증/인가, 제보 CRUD, 담당자/상태/우선순위 저장 |
| 2 | 도움 요청 관리, 경험 피드 검수, 감사 로그 |
| 3 | 공공데이터 배치 동기화, 반복 문제 분석 |
| 4 | PDF/CSV 생성, 파일 업로드, 리포트 이력 |
| 5 | AI 익명화 실제 API 연결, 고급 추천 모델 |
