# ONDA 발표용 구현 상태 표

PPT에 그대로 옮겨 넣기 위한 구현 상태 표입니다.

| 기능 | 상태 | 근거 | 발표 멘트 |
|---|---|---|---|
| 관리자 로그인/내 정보 | 실제 API | Spring Boot auth/session API | 센터/시설팀/슈퍼관리자 계정으로 역할 기반 운영 화면에 접근합니다. |
| 대시보드 KPI | 실제 API | `/api/admin/dashboard/stats` | 제보, 공감, 도움 요청, 미해결 현황을 학교 운영 지표로 보여줍니다. 단, 고위험 제보는 공감/반복 수가 낮아도 우선 검토 대상입니다. |
| 접근성 제보 처리 | 실제 API | 목록/상태/담당자/우선순위/메모/첨부파일 | 학생 제보가 담당자 배정과 조치 흐름으로 이어집니다. |
| 도움 요청 | 실제 API | 목록/상태/센터 판단 | 긴급 도움 요청을 센터 확인 업무로 전환합니다. |
| 경험 피드 | 실제 API + 규칙 기반 | 공개 상태 변경, 규칙 기반 익명화 | 장애학생이 소수라 특정 가능한 상황을 줄이기 위한 검수 구조입니다. |
| 개선 워크플로우 | 실제 API | 단계 변경 | 제보가 검토, 시설팀 전달, 조치 예정, 해결로 이어집니다. |
| 월간 리포트 | 실제 API | PDF/CSV 다운로드, 스냅샷 저장 | 학교에 제출 가능한 개선 근거를 생성합니다. |
| 공공데이터 | 일부 실제 API | 70개 후보, 68개 슬롯, 3개 호출 검증 | 공공데이터는 핵심 판단 근거가 아니라 현장 제보를 보조하는 참고 레이어입니다. |
| 공공데이터 지도 레이어 | MVP 구현 | 원본/정규화 row 기반 후보 레이어 | 좌표가 있는 데이터도 현장 확인 전까지 후보 위치로 보고, 좌표가 없으면 데이터셋 후보로만 표시합니다. |
| 감사 로그 | 실제 API | 조회/CSV 내보내기 | 관리자 처리 이력을 추적할 수 있습니다. |
| 실제 외부 LLM | 추가 예정 | API key/비용/민감정보 검토 필요 | 현재는 규칙 기반 MVP이며, AI는 결정이 아니라 관리자 선택지를 넓히는 보조 기능으로 둡니다. |
| 운영 DB | 추가 예정 | 현재 H2 파일 DB, Flyway 준비 | 운영 전 PostgreSQL/MySQL/Oracle 등으로 전환합니다. |
| 권한별 세부 인가 | 추가 예정 | 역할 모델은 있음 | 장애/위치 민감정보 보호를 위해 API 단위 인가를 고도화합니다. |

## 핵심 메시지

“ONDA는 장애학생의 현장 제보를 중심으로 학교가 개선 우선순위를 검토하게 만들고, 공공데이터와 추천 기능은 그 판단을 보조하는 참고 근거로 사용하는 플랫폼입니다.”
