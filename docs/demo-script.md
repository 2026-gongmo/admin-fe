# ONDA 관리자 웹 발표 시연 스크립트

## 핵심 메시지

ONDA 관리자 웹은 장애학생의 개별 불편을 단순 민원으로 끝내지 않고, 반복 제보와 공감 데이터를 학교 개선 근거로 바꾸는 운영 대시보드입니다.

## 3분 시연 흐름

| 순서 | 화면 | 말할 내용 |
|---:|---|---|
| 1 | `#/dashboard` | "오늘 학교가 먼저 봐야 할 접근성 문제를 KPI와 AI 추천 카드로 확인합니다." |
| 2 | `#/public-data` | "공공데이터에는 접근 가능으로 보이지만 실제 학생 제보에서는 문제가 반복되는 지점을 비교합니다." |
| 3 | `#/reports` | "제보 상세에서 담당자, 우선순위, 처리 메모, 개선 요청서 초안을 확인합니다." |
| 4 | `#/help-requests` | "긴급 도움 요청은 단순 호출 로그가 아니라 이후 시설 개선 제보로 연결될 수 있습니다." |
| 5 | `#/stories` | "장애학생의 경험은 AI 익명화와 관리자 검수를 거쳐 공감 가능한 피드로 관리됩니다." |
| 6 | `#/monthly-report` | "월간 리포트는 학교와 시설팀에 제출할 수 있는 개선 근거 자료가 됩니다." |

## 5분 시연 흐름

1. `#/dashboard`
   - 전체 제보 수, 공감 수, 미해결 건, 도움 요청 수를 보여줍니다.
   - "관리자는 감으로 판단하지 않고 데이터로 우선순위를 봅니다."

2. `#/public-data`
   - 공공데이터와 실제 제보 사이의 차이를 보여줍니다.
   - "공공데이터는 기초 레이어이고, ONDA는 현장성을 보완합니다."

3. `#/reports`
   - 검색바로 특정 건물을 찾습니다.
   - 담당자 배정과 우선순위 변경을 보여줍니다.
   - "현재는 Mock 상태라 새로고침하면 초기화되고, 실제 저장은 아직 구현 안 됨 · 추가 예정입니다."

4. `#/help-requests`
   - 도움 요청 상세 패널을 보여줍니다.
   - 센터 판단, 처리 이력, 연결 후보 제보를 설명합니다.

5. `#/stories`
   - AI 익명화 전/후 비교를 보여줍니다.
   - "개인 경험을 공개 콘텐츠로 바꾸기 전에 개인정보 보호와 검수 흐름을 둡니다."

6. `#/workflow`
   - 제보가 개선 과제로 넘어가는 단계를 보여줍니다.
   - "개별 제보가 시설 개선 업무로 이어지는 흐름입니다."

7. `#/monthly-report`
   - 건물별 리포트, 고위험 미해결 항목, 개선 요청서 미리보기를 보여줍니다.
   - "학교가 예산과 시설 개선 우선순위를 정할 수 있는 자료로 전환됩니다."

8. `#/settings`
   - Mock/API 상태, 권한 정책, 백엔드 체크리스트를 보여줍니다.
   - "프론트는 시연용으로 완성했고, 실제 서비스화를 위해 필요한 백엔드 작업을 분리했습니다."

## 발표 때 조심할 표현

- "연동되어 있습니다"라고 말하지 않습니다.
- "현재는 일부 Spring Boot API가 연결되어 있고, 아직 구현하지 않은 기능은 추가 예정으로 표시합니다"라고 설명합니다.
- "AI가 자동 판단합니다"보다 "AI가 검토 후보와 초안을 보조합니다"라고 설명합니다.
- "공공데이터가 완벽합니다"보다 "공공데이터를 기초로 하고 현장 제보로 보완합니다"라고 설명합니다.
