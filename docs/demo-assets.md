## 발표자료에 넣을 문구

### 한 줄 설명

ONDA 관리자 웹은 학생 앱에서 쌓인 제보, 공감, 도움 요청을 학교가 실제 개선 과제로 관리할 수 있게 만든 운영 대시보드입니다.

### 정확성 문구

현재 화면은 프론트 MVP이며 모든 데이터는 Mock입니다. 실제 저장, 로그인, 공공데이터 동기화, PDF 생성은 Spring Boot 백엔드 연결 후 구현해야 합니다.

### 관리자 웹에서 보여줄 핵심

1. 반복 제보와 공감 수로 개선 우선순위를 정합니다.
2. 공공데이터와 현장 제보의 차이를 보여줍니다.
3. 장애학생 경험 피드는 AI 익명화와 관리자 검수를 거칩니다.
4. 제보는 개선 워크플로우와 월간 리포트로 이어집니다.
5. 실제 서비스화를 위해 필요한 백엔드 작업을 화면과 문서에서 분리했습니다.

## 스크린샷 재생성 방법

```bash
npm run dev -- --host 127.0.0.1
```

브라우저에서 아래 주소를 열고 캡처합니다.

```text
http://127.0.0.1:5173/#/dashboard
http://127.0.0.1:5173/#/reports?selected=r_1
http://127.0.0.1:5173/#/help-requests?selected=h_1
http://127.0.0.1:5173/#/workflow
http://127.0.0.1:5173/#/stories?q=조별과제
http://127.0.0.1:5173/#/demo-guide
```
