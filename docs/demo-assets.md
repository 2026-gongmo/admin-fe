## 발표자료에 넣을 문구

### 한 줄 설명

ONDA 관리자 웹은 학생 앱에서 쌓인 제보, 공감, 도움 요청을 학교가 실제 개선 과제로 관리할 수 있게 만든 운영 대시보드입니다.

### 정확성 문구

현재 화면은 프론트 MVP에 Spring Boot API 주요 기능을 연결한 상태입니다. 로그인, 내 정보, 대시보드 KPI, 제보/도움 요청 관리, 제보 첨부파일, 경험 피드 공개 상태/규칙 기반 익명화, 개선 워크플로우 단계, 월간 리포트 PDF/CSV 다운로드, 공공데이터 seed 조회/샘플/전체 페이지 수집 endpoint, 정규화 요약, 감사 로그 조회/CSV 내보내기를 API로 확인할 수 있습니다. 외부 LLM API, 운영 DB 실연결, 공문/외부 공유 자동 전송은 별도 승인 후 진행합니다.

### 관리자 웹에서 보여줄 핵심

1. 고위험 제보를 먼저 보고, 반복 제보와 공감 수는 보조 근거로 함께 확인합니다.
2. 공공데이터는 현장 제보를 대체하지 않고 참고 레이어로만 보여줍니다.
3. 장애학생 경험 피드는 특정 가능성이 있는 표현을 줄이기 위해 익명화와 관리자 검수를 거칩니다.
4. 제보는 개선 워크플로우와 월간 리포트로 이어집니다.
5. 실제 서비스화를 위해 필요한 백엔드 작업을 화면과 문서에서 분리했습니다.

## 스크린샷 재생성 방법

```bash
npm run dev -- --host 127.0.0.1
```

기본 포트 `5173`이 사용 중이면 `5174`를 사용합니다.

```bash
npm run dev -- --host 127.0.0.1 --port 5174
```

브라우저에서 아래 주소를 열고 캡처합니다. 아래 예시는 `5174` 기준입니다.

```text
http://127.0.0.1:5174/#/dashboard
http://127.0.0.1:5174/#/reports?selected=r_1
http://127.0.0.1:5174/#/help-requests?selected=h_1
http://127.0.0.1:5174/#/workflow?selected=task_1
http://127.0.0.1:5174/#/stories?q=조별과제
http://127.0.0.1:5174/#/monthly-report
http://127.0.0.1:5174/#/settings
http://127.0.0.1:5174/#/demo-guide
```

캡처 전 설정 화면의 Mock API 실패 범위는 `정상` 상태로 돌려둡니다. 실패 UI 자체를 보여줄 때만 `reports`, `helpRequests`, `stories`, `improvementTasks`, `all` 중 하나를 켭니다.
