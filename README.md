# ONDA Admin Web

ONDA 관리자 웹은 장애대학생의 접근성 제보, 공감 데이터, 도움 요청, 공공데이터 비교 결과를 학교/장애학생지원센터가 확인하는 React 기반 대시보드입니다.

현재 버전은 공모전 시연용 프론트엔드 MVP이며, 백엔드는 아직 연결되어 있지 않습니다. 모든 화면 데이터는 `src/data/mockData.ts`의 Mock 데이터를 사용하고, 추후 Spring Boot API 연동 시 `src/services/api.ts`를 교체하는 구조입니다.

## 기술 스택

| 영역 | 기술 |
|---|---|
| Frontend | React, TypeScript, Vite |
| Routing | React Router HashRouter |
| Data | Mock data |
| Styling | CSS |

## 주요 화면

| 경로 | 설명 |
|---|---|
| `#/dashboard` | 전체 KPI, 제보 밀도, AI 우선 조치 추천 |
| `#/reports` | 접근성 제보 목록, 상세 보기, 상태 변경 |
| `#/public-data` | 공공데이터와 현장 제보 비교 |
| `#/analysis` | 반복 문제 분석, AI 추천 조치 |
| `#/help-requests` | 긴급 도움 요청 목록, 상세 패널, 센터 판단, 처리 이력 |
| `#/stories` | 장애학생 경험 피드 관리, AI 익명화/검수 상태 |
| `#/workflow` | 개선 과제 처리 단계 관리 |
| `#/monthly-report` | 월간 접근성 리포트 |
| `#/demo-guide` | 공모전 발표용 시연 순서 안내 |

## 발표 시연 흐름

1. `#/dashboard`에서 오늘의 우선 개선 과제를 확인합니다.
2. `#/public-data`에서 공공데이터와 실제 현장 제보의 차이를 보여줍니다.
3. `#/reports`에서 제보 원문, 공감 수, AI 추천 조치, 공공데이터 연결 정보를 확인합니다.
4. `#/reports`에서 개선 요청서 초안과 관리자 처리 이력을 확인합니다.
5. `#/reports`에서 검색, 담당자 배정, 우선순위 변경 UI를 확인합니다.
6. `#/reports`에서 처리 메모, 상태 변경 사유, 담당 부서, 해결 체크리스트, 첨부파일 Mock을 확인합니다.
7. `#/help-requests`에서 도움 요청 상세, 센터 판단, 접근성 제보 연결 후보를 확인합니다.
8. `#/stories`에서 AI 익명화/민감정보 검수 상태, 원문/공개본 비교, 검색 흐름을 확인합니다.
9. `#/settings`에서 Mock 관리자 로그인, 역할별 접근 정책, 감사 로그 필요성을 확인합니다.
10. `#/settings`에서 화면별 API 매핑표, 백엔드 체크리스트, 로딩/에러/빈 상태 샘플을 확인합니다.
11. `#/workflow`에서 제보가 학교 개선 과제로 넘어가는 상태를 변경해 봅니다.
12. `#/monthly-report`에서 건물별 리포트, 고위험 미해결 항목, 개선 요청서 미리보기, PDF/CSV 내보내기 Mock을 확인합니다.
13. `#/demo-guide`에서 3분/5분 발표 멘트, 평가 항목 매핑, 발표 전 체크리스트, 백엔드 추가 필요 항목을 확인합니다.

## 업로드 전 주의사항

- `node_modules/`는 GitHub에 올리지 않습니다.
- `dist/`는 별도 요청이 없으면 올리지 않습니다.
- `.env`, API Key, Token, 개인정보는 포함하지 않습니다.
- 현재 화면은 Mock 데이터 기반이며 실제 학교 시스템과 연동된 상태가 아닙니다.
